# 函数式语言特性：闭包与迭代器

Auto 的设计借鉴了许多现有语言和技术，其中一个重要的影响来源是_函数式编程_。函数式编程风格通常包括将函数作为值传递、从其他函数返回、赋值给变量以供后续执行等。

本章将介绍：

- _闭包_，可以存储在变量中的匿名函数
- _迭代器_，处理一系列元素的方式
- 如何用闭包和迭代器改进[第 12 章][ch12]的 I/O 项目
- 闭包和迭代器的性能

## 闭包

闭包是可以保存在变量中或作为参数传递给其他函数的匿名函数。与普通函数不同，闭包可以捕获定义它的作用域中的值。

### 闭包语法

Auto 闭包使用 `(params) => expr` 语法：

```auto
let add = (a int, b int) => a + b
let result = add(5, 3)
print(result)
```

```rust
let add = |a: i32, b: i32| a + b;
let result = add(5, 3);
println!("{}", result);
```

这等价于 Rust 的 `|params| expr` 语法。闭包也可以有显式返回类型：

```auto
let expensive_closure = (num int) -> int {
    print("calculating slowly...")
    num
}
```

```rust
let expensive_closure = |num: i32| -> i32 {
    println!("calculating slowly...");
    num
};
```

### 捕获环境

闭包可以从其环境中捕获值。以下是一个使用 `unwrap_or_else` 的示例，其中闭包捕获了 `self`：

<Listing number="13-1" file-name="src/main.rs" caption="闭包从环境中捕获 `self`">

```auto
enum ShirtColor { Red, Blue }

type Inventory {
    shirts []ShirtColor
}

ext Inventory {
    fn giveaway(self, user_preference ?ShirtColor) ShirtColor {
        user_preference.unwrap_or_else((self) => self.most_stocked())
    }

    fn most_stocked(self) ShirtColor {
        var num_red = 0
        var num_blue = 0
        for color in self.shirts {
            color is
                ShirtColor.Red -> num_red += 1
                ShirtColor.Blue -> num_blue += 1
        }
        if num_red > num_blue {
            ShirtColor.Red
        } else {
            ShirtColor.Blue
        }
    }
}

fn main() {
    let store = Inventory(shirts: [ShirtColor.Blue, ShirtColor.Red, ShirtColor.Blue])
    let user_pref1 = ShirtColor.Red
    let giveaway1 = store.giveaway(user_pref1)
    print("The user with preference ${user_pref1} gets ${giveaway1}")

    let user_pref2 = None
    let giveaway2 = store.giveaway(user_pref2)
    print("The user with preference ${user_pref2} gets ${giveaway2}")
}
```

闭包 `(self) => self.most_stocked()` 捕获了 `self` 的不可变引用并调用
`most_stocked()`。`?T`（Option）上的 `unwrap_or_else` 方法仅在值为 `None` 时
才调用闭包。

### 类型推断

闭包通常不需要类型标注——编译器会从使用中推断：

<Listing number="13-2" file-name="src/main.rs" caption="闭包类型推断">

```auto
fn main() {
    let example_closure = (x) => x
    let s = example_closure(String.from("hello"))
    print(s)
}
```

```rust

    let example_closure = |x| x;
    let s = example_closure(String::from("hello"));

```

</Listing>

编译器从第一次调用推断 `x` 的类型为 `String`。一旦推断完成，闭包的类型就被
锁定——用不同的类型调用它会导致编译错误。

### 捕获引用或转移所有权

闭包可以通过三种方式捕获值，对应函数接收参数的三种方式：

1. **不可变引用** — 闭包只读取值
2. **可变引用** — 闭包修改值
3. **所有权** — 闭包获取所有权（Rust 中使用 `move`）

<Listing number="13-3" file-name="src/main.rs" caption="带显式类型标注的闭包">

```auto
fn main() {
    let expensive_closure = (num int) -> int {
        print("calculating slowly...")
        num
    }
    let result = expensive_closure(10)
    print(result)
}
```

### Fn 特征

与 Rust 一样，闭包实现一个或多个 `Fn` 特征：

- **`FnOnce`** — 只能调用一次（移出捕获的值）
- **`FnMut`** — 可以多次调用，可能修改捕获的值
- **`Fn`** — 可以多次调用且不修改捕获的值

这些特征在将闭包传递给函数或存储在类型中时很重要。

## 迭代器

迭代器提供了一种处理元素序列的方式。在 Auto 中，`for` 循环是迭代的主要方式：

```auto
for item in collection {
    print(item)
}
```

```rust
for item in &collection {
    println!("{}", item);
}
```

### `for` 循环

`for` 循环适用于数组、列表、范围以及任何实现迭代器模式的类型：

```auto
// 遍历数组
let arr = [1, 2, 3]
for n in arr {
    print(n)
}

// 遍历范围
for i in 0..10 {
    print(i)
}
```

```rust
let arr = [1, 2, 3];
for n in arr {
    println!("{}", n);
}

for i in 0..10 {
    println!("{}", i);
}
```

在底层，`for` 循环使用迭代器。循环获取迭代器的所有权，重复调用 `next()`
直到返回 `None`。

### 迭代器方法

> **注意**：以下迭代器方法是 Rust `Iterator` 特征的标准方法。当使用 Auto 的
> `use.rust` 时，可以直接在实现 `Iterator` 的类型上访问这些方法。

#### 创建迭代器

| 方法 | 描述 | 所有权 |
|------|------|--------|
| `.iter()` | 遍历引用 | 借用每个元素 |
| `.iter_mut()` | 遍历可变引用 | 可变借用每个元素 |
| `.into_iter()` | 遍历拥有的值 | 获取所有权 |

#### 消费适配器

消费适配器调用 `next()` 并消耗迭代器：

```rust
// sum — 消耗迭代器并返回总和
let v1 = vec![1, 2, 3];
let total: i32 = v1.iter().sum();  // 6
```

#### 迭代器适配器

迭代器适配器转换迭代器但不消耗它。它们是_惰性的_——在消费适配器被调用之前
不会执行任何操作：

```rust
// map — 转换每个元素（惰性！）
let v1 = vec![1, 2, 3];
let v2: Vec<_> = v1.iter().map(|x| x + 1).collect();
// v2 为 [2, 3, 4]

// filter — 保留匹配谓词的元素（惰性！）
let shoes: Vec<Shoe> = shoes.into_iter()
    .filter(|s| s.size == shoe_size)
    .collect();
```

### 在迭代器方法中使用闭包

迭代器方法通常接受闭包作为参数。以下示例使用 `filter` 和一个从环境中捕获
`shoe_size` 的闭包：

```rust
fn shoes_in_size(shoes: Vec<Shoe>, shoe_size: u32) -> Vec<Shoe> {
    shoes.into_iter().filter(|s| s.size == shoe_size).collect()
}
```

闭包 `|s| s.size == shoe_size` 从周围作用域捕获 `shoe_size`，并与每双鞋的
尺寸进行比较。

## 改进 I/O 项目

借助闭包和迭代器，我们可以改进[第 12 章][ch12]的 minigrep 项目。

### 用迭代器重构 `search`

原始的 `search` 函数使用了一个可变的 `results` 向量：

```auto
fn search(query String, contents String) []String {
    var results = List.new()
    for line in contents.lines() {
        if line.contains(query) {
            results.push(line)
        }
    }
    results
}
```

使用迭代器适配器，我们可以使代码更简洁，避免可变中间变量：

```rust
pub fn search<'a>(query: &str, contents: &'a str) -> Vec<&'a str> {
    contents
        .lines()
        .filter(|line| line.contains(query))
        .collect()
}
```

这里使用 `filter` 只保留包含查询字符串的行，然后用 `collect` 将它们收集到
向量中。函数式风格最小化了可变状态，使代码更易理解。

## 闭包与迭代器速查

| 特性 | Auto | Rust |
|------|------|------|
| 闭包语法 | `(x) => x + 1` | `\|x\| x + 1` |
| 带类型的闭包 | `(x int) -> int { x + 1 }` | `\|x: i32\| -> i32 { x + 1 }` |
| for 循环 | `for x in list { ... }` | `for x in &list { ... }` |
| 范围 | `0..10` | `0..10` |
| 过滤 | `.filter((s) => s.size == 10)` | `.filter(\|s\| s.size == 10)` |
| 映射 | `.map((x) => x + 1)` | `.map(\|x\| x + 1)` |
| 收集 | `.collect()` | `.collect()` |
| 求和 | `.sum()` | `.sum()` |

## 总结

本章介绍了两个关键的函数式编程特性：

1. **闭包** — 捕获其环境的匿名函数
   - `(params) => expr` 语法
   - 从使用中推断类型
   - 通过不可变引用、可变引用或所有权捕获
   - `Fn`、`FnMut`、`FnOnce` 特征

2. **迭代器** — 逐个处理元素的惰性序列
   - `for` 循环作为主要迭代机制
   - 消费适配器（`.sum()`）和迭代器适配器（`.map()`、`.filter()`）
   - 链式适配器实现复杂转换

3. **I/O 项目改进** — 用迭代器适配器简化 `search`

闭包和迭代器在 Rust 中是零成本抽象——它们编译为与手写循环相同的代码。
Auto 继承了这一性能模型。
