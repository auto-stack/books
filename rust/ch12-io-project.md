## Accepting Command Line Arguments

To read command line arguments, use `env.args()`:

<Listing number="12-1" file-name="src/main.rs" caption="Reading command line arguments with `env.args()`">

```auto
use env

fn main() {
    let args = env.args()
    print(f"Args: ${args}")
}
```

```rust

    println!("Args: {:?}", args);
}
```

</Listing>

`env.args()` returns an iterator of the command line arguments. The first
argument is always the program name itself. To collect them into a `List`:

```auto
let args = env.args().collect()
```

> Note: `env.args()` will panic if any argument contains invalid Unicode.
> Use `env.args_os()` for arguments that may contain invalid Unicode.

## Reading a File

Use `fs.read_to_string()` from Rust's standard library via `use.rust`:

<Listing number="12-2" file-name="src/main.rs" caption="Reading a file specified by command line arguments">

```auto
use.rust std::fs

fn main() ! {
    let args = env.args()
    if args.len() < 3 {
        panic("not enough arguments")
    }
    let query = args[1]
    let file_path = args[2]

    let contents = fs.read_to_string(file_path).!
    print(contents)
}
```

```rust
// Auto-generated by a2r transpiler

// a2r Standard Library (from crate)
#[allow(unused_imports)]
use auto_lang::a2r_std::*;

use std::fs;

fn main() -> Result<(), String> {
    let args: Vec<String> = env::args().collect();
    if args.len() < 3 {
        panic!("not enough arguments");
    }
    let query = &args[1];
    let file_path = &args[2];

    let contents = fs::read_to_string(file_path).collect();
    println!("{}", contents);
    Ok(())
}
```

</Listing>

Key points:

- `fn main() !` — the `!` return type enables error propagation
- `fs.read_to_string(file_path).!` — reads the file, propagating errors via `.collect()`
- `use env` and `use.rust std::fs` — import environment and filesystem modules

## Refactoring for Modularity

As the program grows, we extract logic into separate functions and a `Config`
type. This follows the Rust convention of separating concerns between binary
(`main.rs`) and library (`lib.rs`) code.

### Extracting a Config Type

<Listing number="12-3" file-name="src/lib.rs" caption="Modular minigrep with Config type and search function">

```auto
type Config {
    query String
    file_path String
}

ext Config {
    fn build(args []String) !Config {
        if args.len() < 3 {
            throw "not enough arguments"
        }
        Config(query: args[1], file_path: args[2])
    }
}

fn search(query String, contents String) []String {
    var results = List.new()
    for line in contents.lines() {
        if line.contains(query) {
            results.push(line)
        }
    }
    results
}

fn run(config Config) ! {
    let contents = fs.read_to_string(config.file_path).!
    for line in search(config.query, contents) {
        print(line)
    }
}

use env
use.rust std::fs

fn main() ! {
    let config = Config.build(env.args())
    run(config)
}
```

```rust
// Auto-generated by a2r transpiler

// a2r Standard Library (from crate)
#[allow(unused_imports)]
use auto_lang::a2r_std::*;

struct Config {
    query: String,
    file_path: String,
}

impl Config {
    fn build(args: Vec<String>) -> Result<Config, String> {
        if args.len() < 3 {
            Err(String::from("not enough arguments"));
        } else {
            Ok(Config { query: args[1].clone(), file_path: args[2].clone() })
        }
    }
}

fn search(query: &str, contents: &str) -> Vec<&str> {
    let mut results = Vec::new();
    for line in contents.lines() {
        if line.contains(query) {
            results.push(line);
        }
    }
    results
}

fn run(config: Config) -> Result<(), String> {
    let contents = fs::read_to_string(&config.file_path).collect();
    for line in search(&config.query, &contents) {
        println!("{}", line);
    }
    Ok(())
}

use std::env;
use std::fs;

fn main() -> Result<(), String> {
    let config = Config::build(env::args().collect())?;
    run(config)
}
```

</Listing>

This refactoring demonstrates several important patterns:

1. **`type Config`** — groups configuration values into a single type
2. **`Config.build()` returns `!Config`** — uses `throw` for error cases
3. **`fn run(config Config) !`** — separates program logic from setup
4. **`fn main() !`** — is minimal: build config, run, handle errors
5. **`.collect()`** on `!T` — equivalent to Rust's `?` operator

## Environment Variables

Environment variables let users configure behavior without changing command
line arguments. Use `env.var("VAR_NAME")` which returns `!String`:

<Listing number="12-4" file-name="src/main.rs" caption="Environment variable for case-insensitive search">

```auto
fn search_case_insensitive(query String, contents String) []String {
    var results = List.new()
    let query_lower = query.to_lowercase()
    for line in contents.lines() {
        if line.to_lowercase().contains(query_lower) {
            results.push(line)
        }
    }
    results
}

fn main() ! {
    let args = env.args()
    let query = args[1]
    let file_path = args[2]
    let ignore_case = env.var("IGNORE_CASE").is_ok()

    let contents = fs.read_to_string(file_path).!

    if ignore_case {
        for line in search_case_insensitive(query, contents) {
            print(line)
        }
    } else {
        for line in search(query, contents) {
            print(line)
        }
    }
}

use env
use.rust std::fs
```

`env.var("IGNORE_CASE")` returns `!String`. The `.is_ok()` method checks if the
environment variable is set (regardless of its value), returning a `bool`.

Run with the environment variable set:

```console
$ IGNORE_CASE=1 automan run -- to poem.txt
```

## Printing to Standard Error

Use `eprint()` for error messages (equivalent to Rust's `eprintln!`):

```auto
eprint("Error: file not found")
```

This sends output to `stderr` instead of `stdout`, so users can redirect
successful output to a file while still seeing error messages on screen:

```console
$ automan run -- query file.txt > results.txt
```

## I/O Quick Reference

| Operation | Auto | Rust |
|-----------|------|------|
| CLI arguments | `env.args()` | `std::env::args()` |
| Read file | `fs.read_to_string(path).!` | `std::fs::read_to_string(path)?` |
| Environment var | `env.var("KEY")` | `std::env::var("KEY")` |
| Check env set | `env.var("KEY").is_ok()` | `std::env::var("KEY").is_ok()` |
| Stdout | `print()` | `println!()` |
| Stderr | `eprint()` | `eprintln!()` |
| Exit with code | `sys.exit(code)` | `std::process::exit(code)` |
