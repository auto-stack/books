# 第一步

现在我们将看到如何在 Auto 中运行传统的"Hello World"程序。这将教会你如何编写、保存、构建和运行 Auto 程序。

与 Python 不同，Auto 使用编译-运行的工作流：你将源代码写在 `.at` 文件中，使用 `auto b` 命令通过 `a2p` 转译器将其转译为 Python，然后运行生成的 Python 文件。让我们来探索这是如何工作的。

## 使用命令行

在操作系统中打开终端（如[安装](ch02-installation.cn.md)一章中所述），导航到你想要存放 Auto 项目的目录。例如：

```console
$ mkdir ~/auto-projects
$ cd ~/auto-projects
```

你可以通过运行以下命令来验证 Auto 工具链是否就绪：

```console
$ autoc --version
```

你应该能看到版本号打印到屏幕上，确认一切安装正确。

### 如何退出

没有需要退出的交互式解释器——Auto 是编译型语言，不是 REPL。如果运行中的程序卡住了，你可以在终端中按 `Ctrl + C` 来中断它。要退出终端本身，输入 `exit` 或按 `Ctrl + D`（在 Linux 和 macOS 上）。

## 选择编辑器

我们需要一个编辑器来编写和保存 Auto 源文件。一个好的程序员编辑器会让你的工作轻松很多。选择编辑器就像选择你要买的汽车一样。一个好的编辑器能帮助你轻松地编写 Auto 程序，让你的旅程更加舒适，并帮助你更快、更安全地到达目的地（实现你的目标）。

最基本的要求之一是*语法高亮*——你的 Auto 程序的不同部分会被着色，这样你就能*看到*你的程序并想象它的运行过程。

以下是一些建议：

- **VS Code** 配合 Auto 扩展——这是最适合初学者的选择。VS Code 是免费的，在 Windows、macOS 和 Linux 上都可用，并提供出色的 Auto 支持，包括语法高亮和 IntelliSense。
- **Vim** 或 **Neovim**——一个强大的、基于终端的编辑器，深受经验丰富的程序员喜爱。如果你已经了解 Vim，你会感到非常顺手。
- **Emacs**——另一个拥有悠久历史的强大编辑器。Emacs 用户通常对自己的编辑器充满热情。

如果你不知道从哪里开始，我推荐使用**VS Code** 配合 Auto 扩展。

> **注意：** 无论你选择哪种编辑器，确保它支持 Auto 语法高亮。这将帮助你在输入时更容易地发现错误。

## 使用源文件

现在让我们回到编程。有一个传统：每当你学习一门新的编程语言时，你编写和运行的第一个程序就是"Hello World"程序——它所做的就是运行时打印"Hello World"。正如 Simon Cozens[^1] 所说，这是"向编程之神祈求帮助你更好地学习语言的传统咒语。"

启动你选择的编辑器，输入以下程序并将其保存为 `hello.at`。

<span class="filename">文件名：hello.at</span>

<Listing number="3-1" file-name="hello.at" caption="你的第一个 Auto 程序">

```auto
fn main() {
    print("Hello, World!")
}
```

```python
# Auto-generated by a2p transpiler

def main():
    print("Hello, World!")

if __name__ == "__main__":
    main()
```

</Listing>

文件应该保存在哪里？保存在你知道路径的任何文件夹中。如果你不明白这是什么意思，创建一个新文件夹，并用那个位置来保存和运行你的所有 Auto 程序：

- macOS 和 Linux 上：`~/auto-projects`
- Windows 上：`C:\auto-projects`

> **注意：** 始终确保你的 Auto 文件使用 `.at` 扩展名，例如 `hello.at`。

### 构建和运行

要构建和运行你的 Auto 程序：

1. 打开一个终端窗口（参见前面[安装](ch02-installation.cn.md)一章中关于如何操作的说明）
2. **切**换到保存文件的目录，例如 `cd ~/auto-projects`
3. 在同一目录下创建一个 `pac.at` 包文件（或使用 `auto init` 命令来生成一个）
4. 输入 `auto b` 运行构建命令。这将把你的 Auto 代码转译为 Python，然后运行它。

```console
$ auto b
```

你应该能看到以下输出：

```text
Hello, World!
```

如果你看到了如上所示的输出，恭喜你！——你已经成功运行了你的第一个 Auto 程序。你已经成功跨越了学习编程最难的部分，即用你的第一个程序开始编程之旅！

如果你遇到了错误，请完全按照上面所示输入程序，然后再次运行构建命令。注意 Auto 是区分大小写的，即 `print` 和 `Print` 是不同的——注意前者是小写 `p`，后者是大写 `P`。

**工作原理**

Auto 程序必须有一个 `fn main()` 函数作为入口点。`a2p` 转译器将其转换为 Python 的 `def main():` 函数，并自动添加标准的 `if __name__ == "__main__": main()` 守卫。在 `main` 内部，我们调用 `print` 函数并向其提供文本 `"Hello, World!"`，然后该文本被打印到屏幕上。

## 获取帮助

如果你需要关于任何 Auto 工具或命令的快速信息，可以使用内置的 `--help` 标志。例如：

```console
$ autoc --help
```

这将显示 `autoc` 编译器的使用信息，包括所有可用的选项和标志。

```console
$ automan --help
```

这将显示 `automan` 包管理器的帮助信息。

```console
$ a2p --help
```

这将显示 `a2p` 转译器的帮助信息，包括控制转译过程的各种选项。

如需详细文档，你可以访问 Auto 官方网站 <https://auto.dev>。

## 小结

你现在应该能够轻松地编写、保存、构建和运行 Auto 程序了。
