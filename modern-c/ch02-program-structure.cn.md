# 程序结构

C 程序由词法单元构成，组织为声明和语句，并按特定顺序编译。Auto 简化了这一结构，但保留了相同的逻辑组织。本章逐步讲解两种语言的文法、声明、定义和语句。

## 2.1 文法

C 程序由*词法单元（token）*构成：标识符、关键字、字面量、运算符和标点符号。C 标准定义了精确的文法，决定哪些词法单元序列构成有效程序。

C 中基本的词法单元类别如下：

- **关键字：** `int`、`double`、`if`、`for`、`while`、`return`、`struct`、`switch`、`case`、`break`、`default`、`void`、`const`……
- **标识符：** 你选择的名字——`main`、`count`、`my_function`、`Point`
- **字面量：** `42`、`3.14`、`"hello"`、`true`
- **运算符：** `+`、`-`、`*`、`/`、`=`、`==`、`<`、`>`、`&&`、`||`……
- **标点符号：** `(`、`)`、`{`、`}`、`;`、`,`

C 的文法是类型前置的。声明时类型在名称之前：

```c
int x = 5;
double pi = 3.14159;
const char* greeting = "hello";
```

Auto 的文法更简洁。类型出现在名称*之后*，需要的关键字更少：

```auto
let x int = 5
let pi float = 3.14159
let greeting str = "hello"
```

### 翻译单元和作用域

C 程序由一个或多个*翻译单元*组成——独立编译然后链接在一起的源文件。每个翻译单元分阶段处理：预处理、词法分析、解析和代码生成。

Auto 遵循相同的模型。每个 `.at` 文件是一个翻译单元。a2c 转译器独立处理每个文件，生成 `.c` 文件供 C 编译器编译。

C 有四种标识符作用域：

- **块作用域：** 在 `{ ... }` 内可见（最常见）。
- **文件作用域：** 在整个翻译单元内可见。
- **函数作用域：** 仅用于标签（很少使用）。
- **函数原型作用域：** 仅在函数声明内。

Auto 有两种实际作用域：函数作用域（在 `fn` 内部）和模块作用域（在顶层）。映射关系很直接。

> **要点：** C 的文法把类型放前面；Auto 的文法把名字放前面。两者携带相同的信息，但 Auto 的顺序在英语中读起来更自然："let x be an int equal to 5."

## 2.2 声明

C 中的*声明*引入一个名称并将其与类型关联。最常见的形式是变量声明：

```c
int age = 25;
double height = 1.75;
char initial = 'A';
bool active = true;
```

在 Auto 中，`let` 关键字引入不可变绑定，`var` 引入可变绑定：

```auto
let age int = 25       // 不可变
let height float = 1.75
var count int = 0      // 可变
```

### 类型推断

Auto 支持使用 `var` 的有限形式类型推断：

```auto
var x = 5              // 推断为 int
var pi = 3.14159       // 推断为 float
var name = "hello"     // 推断为 str
```

a2c 转译器从初始化器推断类型并生成相应的 C 类型。C 的 `auto` 关键字（C23 中添加）提供了类似功能，但 Auto 的 `var` 在所有目标 C 标准中都能工作，因为推断发生在转译时。

### const 和 let

C 的 `const` 限定符将变量标记为初始化后只读：

```c
const int x = 5;      // 不能修改
x = 10;               // 编译错误
```

Auto 的 `let` 通过约定提供相同的保证——`let` 绑定不能被重新赋值。a2c 转译器不会为 `let` 绑定生成 `const`（因为 C 的 `const` 在某些上下文中有不同语义），但意图已经在源代码中记录了。

> **C 深潜：** C 区分*声明*（宣布名称和类型）与*定义*（分配存储）。大多数情况下声明也是定义。`extern` 关键字可以创建不是定义的声明，引用在别处定义的变量。Auto 没有 `extern` 的等价物——所有绑定都是定义。

> **要点：** 用 `let` 表示不会改变的值，用 `var` 表示会改变的值。a2c 转译器将两者都映射为 C 变量声明，但这个区别记录了你的意图。

## 2.3 定义

在 C 中，*定义*创建一个实体。分配存储时变量被定义。提供函数体时函数被定义。

```c
// 变量定义
int counter = 0;

// 函数定义
int square(int x) {
    return x * x;
}
```

在 Auto 中，相同的概念以更简洁的语法呈现：

```auto
// 变量定义
var counter int = 0

// 函数定义
fn square(x int) int {
    x * x
}
```

注意差异：

- Auto 函数对最后一个表达式省略 `return`——其值就是返回值。
- Auto 使用 `fn` 而非 C 的返回类型前置语法。
- 参数遵循 name:type 约定。

### 对象和值

C 使用*对象*这个词表示持有值的内存区域。一个对象具有：

- 一个*类型*（持有什么样的值）。
- 一个*生命周期*（内存何时有效）。
- 一个*存储期*（自动、静态、分配或线程）。

Auto 在底层使用相同的模型。当你写 `let x int = 5` 时，a2c 转译器创建一个具有自动存储期的 `int` 对象。值 `5` 被存储在该对象中。

> **C 深潜：** C 有*暂定定义*的概念——没有初始化器的文件作用域声明是否为定义取决于上下文。这是微妙 bug 的来源。Auto 完全避免了这一点：任何作用域的每个 `let` 或 `var` 都是完整定义。

> **要点：** 每个 Auto 绑定都是定义。没有暂定的或部分的声明。

## 2.4 语句

*语句*是程序执行的指令。C 有以下几种：

- **表达式语句：** `x = 5;`——表达式后加分号。
- **迭代语句：** `for`、`while`、`do...while`。
- **选择语句：** `if`、`switch`。
- **跳转语句：** `return`、`break`、`continue`、`goto`。

Auto 提供相同的类别，语法更简洁：

- **表达式语句：** `x = 5`——不需要分号。
- **迭代：** `for i in 0..n { }`、`while cond { }`。
- **选择：** `if cond { }`、`is x { ... }`。
- **跳转：** `return`（通常是隐式的）。


变为：
```c
count = count + 1;
```

以及：

```auto
for i in 0..5 {
    count = count + 1
}
```
变为：
```c
for (int i = 0; i < 5; i++) {
    count = count + 1;
}
```

### 复合语句

在 C 中，*复合语句*（或块）是一对花括号，包围零个或多个声明和语句：

```c
{
    int x = 5;
    x = x + 1;
    printf("%d\n", x);
}
```

Auto 使用相同的花括号分隔块：

```auto
{
    let x int = 5
    var y int = x + 1
    print(y)
}
```

关键区别：Auto 不需要分号。每条语句占据自己的行（或以换行符分隔）。这消除了整类语法错误。

> **C 深潜：** 在 C 中，每条语句必须以分号（`;`）结尾。分号是语句终止符，不是分隔符。忘记分号是最常见的 C 编译错误之一。Auto 完全消除了这一点——换行符作为语句边界。

> **要点：** 语句是命令式编程的构建块。Auto 简化了语句语法，同时保留了 C 的执行模型。

## 快速参考

| 概念 | C | Auto |
|---|---|---|
| 不可变绑定 | `const int x = 5;` | `let x int = 5` |
| 可变绑定 | `int x = 5;` | `var x int = 5` |
| 类型推断 | `auto x = 5;`（C23） | `var x = 5` |
| 字符串类型 | `const char*` | `str` |
| 浮点类型 | `double` | `float` |
| 布尔类型 | `bool` | `bool` |
| 函数 | `int f(int x) { return x; }` | `fn f(x int) int { x }` |
| 表达式语句 | `x = 5;` | `x = 5` |
| 复合语句 | `{ ... }` | `{ ... }` |
| 分号 | 必需 | 不使用 |
