# 第四章：表达式

> 级别 1 — 入门
>
> C 和 Auto 如何通过运算符和求值来组合值。

表达式是任何程序的主力。每一次计算——从两个数相加到决定走哪个分支——最终都归结为求值一个表达式。C 提供了丰富的运算符集；Auto 保持了相同的能力，但以更安全、更可读的形式呈现。

---

## 4.1 操作数与运算符

**操作数**是运算符作用的值。**运算符**指定对其操作数执行什么操作。C 按操作数数量对运算符进行分类：

| 元数         | C 示例    | 含义                 |
|-------------|-----------|---------------------|
| 一元        | `-x`      | 取反                 |
| 二元        | `a + b`   | 加法                 |
| 三元        | `a ? b : c` | 条件选择           |

Auto 使用相同的分类，但用 `if/else` 表达式语法替代了三元运算符 `?:`。下表将最常见的 C 运算符映射到 Auto 等价物。

| 类别             | C 运算符      | Auto 等价            | 备注                         |
|-----------------|---------------|---------------------|------------------------------|
| 算术             | `+ - * / %`  | `+ - * / %`         | 相同                         |
| 自增/自减        | `++ --`      | `x = x + 1`         | 无副作用运算符               |
| 复合赋值         | `+= -= *=`   | `x = x + y`         | 显式赋值                     |
| 比较             | `== != < > <= >=` | `== != < > <= >=` | 相同                    |
| 逻辑             | `&& \|\| !`  | `and or not`        | Auto 使用单词形式            |
| 三元             | `a ? b : c`  | `if a { b } else { c }` | 表达式形式              |
| 位运算           | `& \| ^ ~ << >>` | 相同运算符        | 两种语言都可用               |

> **要点：** Auto 移除了就地修改对象的运算符（`++`、`--`、`+=`），用普通赋值替代。这使得每个表达式的效果都是显式的。

## 4.2 算术运算

五个算术运算符在 C 和 Auto 中完全相同：

```
+  加法          -  减法
*  乘法          /  除法         %  取余（取模）
```

**整数除法**在两种语言中都向零截断。`17 / 5` 得到 `3`，而不是 `3.4`。取余运算符 `%` 给出余数：`17 % 5` 得到 `2`。

Auto 的 `print` 命令自动处理输出格式，不像 C 的 `printf` 需要格式说明符如 `%d` 和 `%f`。

```c
// C
printf("a + b = %d\n", a + b);

// Auto
print("a + b =", a + b)
```

## 4.3 修改对象的运算符

C 提供了将运算与赋值结合的运算符：

```c
count++;        // count = count + 1
count += 3;     // count = count + 3
total -= discount;  // total = total - discount
```

Auto **没有** `++`、`--` 或复合赋值运算符。你需要写出完整的赋值：

```auto
count = count + 1
count = count + 3
total = total - discount
```

这是一个有意的设计选择。在 C 中，自增运算符 `++` 可以作为前缀或后缀出现，表达式的*值*取决于你使用哪种形式：

```c
int a = 5;
int b = a++;   // b = 5, a 变为 6（后缀）
int c = ++a;   // c = 7, a 变为 7（前缀）
```

这种双重性质——在同一个表达式中产生一个值*并*修改一个对象——是常见的 bug 来源。Auto 通过将读取和写入分离来消除了这个问题。

> **要点：** 如果你需要递增，写 `x = x + 1`。它明确表达了它的意图，没有求值顺序的歧义。

## 4.4 布尔上下文

在 C 中，任何标量值都可以在布尔上下文中测试：

- **零值**（整数 `0`、浮点 `0.0`、空指针 `NULL`）为**假**。
- **其他所有值**为**真**。

```c
if (x) { /* x 非零 */ }
if (!y) { /* y 为零 */ }
```

Auto 引入了真正的 `bool` 类型，值为 `true` 和 `false`。真值测试是相同的——类零值为假——但 Auto 使类型显式化：

```auto
let flag bool = true
if flag {
    print("flag is set")
}
```

> **要点（C 深入）：** 永远不要显式地与 `0` 或 `1` 进行布尔测试比较。写 `if (ptr)` 而不是 `if (ptr != NULL)`，写 `if (!done)` 而不是 `if (done == 0)`。隐式布尔测试是惯用 C，读起来更自然。

## 4.5 三元运算符

C 的条件表达式使用 `? :` 运算符：

```c
int max = (a > b) ? a : b;
const char *label = (x > 0) ? "positive" : "non-positive";
```

Auto 用返回值的 `if/else` 表达式替代：

```auto
let max int = if a > b { a } else { b }
let label str = if x > 0 { "positive" } else { "non-positive" }
```

Auto 的形式更冗长但更清晰：两个分支用花括号视觉上分离，没有晦涩的 `? :` 标点需要解读。

> **要点：** 三元运算符是 C 中*唯一*的三元运算符。Auto 泛化了这个概念：任何 `if/else` 块在值位置使用时都可以作为表达式。

## 4.6 求值顺序

> **C 深入：** 求值顺序是 C 最棘手的方面之一。语言标准区分了*优先级*（哪个运算符绑定更紧）和*顺序*（哪个操作数先求值）。

**优先级**决定分组：`a + b * c` 被解析为 `a + (b * c)`，因为 `*` 的优先级高于 `+`。两种语言相同。

**顺序**决定副作用*何时*发生。在 C 中，操作数的求值顺序在很大程度上是**未指定的**。考虑：

```c
int a = 3;
int b = a++ + a++;   // 未定义行为
```

两个对 `a` 的修改之间没有*序列点*。编译器可以自由选择先求值左边的 `a++`、右边的 `a++`，或任何其他顺序。结果是不可预测的。

Auto 完全避免了这类 bug，因为它没有副作用运算符。你不能在子表达式中修改变量。

**C 中的序列点**出现在：
- `&&` 和 `||` 运算符（左操作数完全求值后）
- `?:` 运算符（条件求值后，恰好一个分支）
- 逗号运算符（左操作数求值并丢弃）
- 完整表达式结束时（`;`）
- 函数调用入口和返回

> **要点：** 在 C 中，永远不要在序列点之间两次修改同一变量。在 Auto 中，你根本做不到——语言阻止了它。
