# 第五章：基本值与数据

> 级别 1 — 入门
>
> 所有计算的基础：类型、值，以及 C 和 Auto 如何表示数据。

这是级别 1 中最重要的章节。你在 C 或 Auto 中构建的一切都建立在对语言如何在内存中表示值的理解之上。我们将涵盖抽象机器模型、基本类型、转换、初始化器、命名常量和二进制表示。

---

## 5.1 抽象状态机

> **C 深入：** C 标准并不描述一台物理计算机。相反，它定义了一个**抽象状态机**，规定了符合标准的程序必须做什么。真实编译器的工作是生成代码，使其表现得*就像*抽象状态机执行了该程序一样。

抽象状态机包括：

- **对象**——保存值的存储区域（变量、分配的内存）。
- **值**——存储在对象中的位模式，根据其类型进行解释。
- **副作用**——机器状态的改变（写入变量、调用 I/O 函数）。
- **序列点**——保证所有先前副作用完成的时刻。

当你写 `int x = 42;` 时，抽象状态机：

1. 为名为 `x` 的对象分配存储空间。
2. 确定其类型为 `int`。
3. 将值 `42` 存储在该对象中。

Auto 通过移除未定义行为简化了这个模型。如果 Auto 程序能编译，它就有明确定义的语义。代价是 Auto 隐藏了 C 暴露的某些机器级细节（如原始指针算术和精确的对象布局）。

> **要点：** C 的抽象状态机是你和编译器之间的*契约*。遵守契约，你的代码就是可移植和可预测的。违反它（例如通过未定义行为），编译器可以做任何事。

## 5.2 基本类型

C 有一组小但精细的基本类型。Auto 将它们映射到更清晰的子集：

| C 类型         | Auto 类型   | 典型大小       | 备注                         |
|----------------|-------------|---------------|------------------------------|
| `char`         | `char`      | 1 字节        | 字符或小整数                  |
| `signed char`  | `i8`        | 1 字节        | 显式有符号 8 位               |
| `unsigned char`| `u8`        | 1 字节        | 无符号 8 位                   |
| `short`        | `i16`       | 2 字节        | 短整数                       |
| `int`          | `int`       | 4 字节        | 默认整数                     |
| `long`         | `i64`       | 4 或 8 字节   | 平台相关                     |
| `long long`    | `i64`       | 8 字节        | 至少 64 位                    |
| `float`        | `float`     | 4 字节        | 单精度 IEEE 754              |
| `double`       | `float`     | 8 字节        | 双精度；Auto 用 `float` 统一表示 |
| `_Bool`        | `bool`      | 1 字节        | C99 布尔；Auto 有 `bool`     |
| `size_t`       | `usize`     | 4 或 8 字节   | 无符号，sizeof 结果          |

主要区别：

- **C 的 `double` 与 `float` 是不同的类型。** Auto 将它们统一为 `float`，让编译器选择精度。
- **C 的每种整数类型都有有符号/无符号变体。** Auto 默认有符号，提供显式的 `u8`、`u16`、`u32`、`u64` 用于无符号需求。
- **C 的 `char` 可能是有符号或无符号的**，取决于平台。Auto 将 `char` 视为仅用于字符数据的独立类型。

## 5.3 指定值

**字面量**是直接写在源代码中的值：

```c
// C 整数字面量
42          // 十进制 int
052         // 八进制 int（十进制 42）
0x2A        // 十六进制 int（十进制 42）
42U         // unsigned int
42LL        // long long

// C 浮点字面量
3.14        // double
3.14f       // float
1.0e-3      // 科学计数法 double
```

Auto 的字面量语法类似，但没有类型后缀：

```auto
let n int = 42          // 整数字面量
let f float = 3.14      // 浮点字面量
let c char = 'A'        // 字符字面量
let s str = "hello"     // 字符串字面量
```

**枚举**创建命名的整数常量：

```c
// C
enum color { RED, GREEN, BLUE };    // 0, 1, 2
enum color fav = GREEN;
```

```auto
// Auto
enum Color {
    Red
    Green
    Blue
}
let fav Color = Color.Green
```

> **要点：** C 的枚举是纯整数——你可以将任何 `int` 赋给 `enum` 变量。Auto 的枚举是具有编译时检查的真正类型。

## 5.4 隐式转换

C 自动执行许多转换。这很方便但很危险：

- **整数提升：** 表达式中的 `char` 和 `short` 被提升为 `int`。
- **算术转换：** 混合类型时，较窄的类型转换为较宽的类型。`int + float` 先将 `int` 提升为 `float`。
- **赋值转换：** 将较宽类型赋给较窄类型会静默截断。`double` 到 `int` 丢弃小数部分。

```c
int i = 3.14;     // i 变为 3 —— 静默截断
double d = 42;    // d 变为 42.0 —— 安全提升
```

Auto 更严格。收窄转换需要显式转换：

```auto
let i int = int(3.14)       // 显式——你确认了截断
let f float = float(42)     // 宽化——可以隐式，显式更清晰
```

> **要点：** 在 C 中，隐式转换可能静默丢失数据。Auto 使收窄转换变为显式，让你思考它们。

## 5.5 初始化器

C 的初始化器语法有了显著发展。现代 C（C99 及以后）支持**指定初始化器**，可以命名要设置的字段：

```c
// C 数组初始化器
int primes[] = { 2, 3, 5, 7, 11 };

// C 结构体初始化器（指定）
struct Point { int x; int y; };
struct Point p = { .x = 1, .y = 2 };
```

Auto 使用**构造函数调用**语法，模仿函数调用：

```auto
// Auto 数组初始化器
let primes [5]int = [5]int{2, 3, 5, 7, 11}

// Auto 结构体初始化器（构造函数语法）
let p Point = Point(1, 2)
```

C 中未初始化的局部变量有**不确定的值**——读取它们是未定义行为。Auto 要求 `let` 绑定显式初始化；`var` 声明可以不初始化，但编译器会警告赋值前使用。

## 5.6 命名常量

C 有三种命名常量机制，每种属性不同：

| 机制           | 示例                   | 类型安全 | 有作用域 | 编译时 |
|---------------|------------------------|---------|---------|--------|
| `#define`     | `#define PI 3.14`      | 否      | 否      | 是     |
| `const`       | `const double PI = 3.14;` | 部分 | 是      | 不定   |
| `enum`        | `enum { MAX = 1024 };` | 是（仅 int）| 是  | 是     |

Auto 用 `let` 统一了这些：

```auto
let pi float = 3.14159       // 不可变常量
let max_size int = 1024      // 不可变常量
```

Auto 的 `let` 总是不可变的，总是有作用域的。编译器在有益时可以在编译时内联这些值。

<Listing path="listings/ch05/listing-05-03" title="命名常量与枚举" />

> **要点：** 在 C 中，整数常量优先使用 `enum`，其他类型用 `const`。避免用 `#define` 做常量——预处理器是个粗糙的工具。在 Auto 中，`let` 统一处理所有情况。

## 5.7 二进制表示

> **C 深入：** 理解值如何以位的形式存储对于系统编程至关重要。C 让你直接访问位级别；Auto 提供库函数进行相同操作。

**二进制补码**是有符号整数的标准表示（C23 强制要求，实际上从 C99 起就已普及）。最高有效位作为符号位：`0` 表示正数，`1` 表示负数。要取反一个值，反转所有位并加 1。

```
值      8 位二进制补码
  0      00000000
  1      00000001
  42     00101010
 127     01111111
 -1      11111111
 -42     11010110
-128     10000000
```

**位运算符**操作单个位：

| 运算符 | C / Auto   | 含义                  | 示例（8 位）              |
|--------|------------|----------------------|--------------------------|
| AND    | `a & b`    | 两位都为 1 → 1       | `0x0F & 0x55 = 0x05`   |
| OR     | `a \| b`   | 任一位为 1 → 1       | `0x0F \| 0x55 = 0x5F`  |
| XOR    | `a ^ b`    | 位不同 → 1           | `0x0F ^ 0x55 = 0x5A`   |
| NOT    | `~a`       | 翻转所有位            | `~0x0F = 0xF0`         |
| 左移   | `a << n`   | 位左移               | `1 << 3 = 8`           |
| 右移   | `a >> n`   | 位右移               | `8 >> 3 = 1`           |

常见位操作技巧：

```c
// 设置第 n 位：   flags |= (1 << n);
// 清除第 n 位：   flags &= ~(1 << n);
// 翻转第 n 位：   flags ^= (1 << n);
// 测试第 n 位：   (flags >> n) & 1
```

> **要点：** 位运算在两种语言中是相同的。Auto 为底层工作提供了这些操作，但大多数程序应使用更高级的抽象。
