# 项目：猜数字游戏

这是第一阶段（Phase 1）的总结项目。在前四章中，你已经学习了变量、运算符、函数、集合和控制流。现在到了把所有这些技能综合起来的时候了——我们将构建一个完整的交互式程序：猜数字游戏。

游戏规则如下：程序在 1 到 100 之间选一个秘密数字，玩家输入猜测值，程序告诉玩家是"太大"还是"太小"，直到猜对为止。我们将分四个阶段来构建这个游戏，每次增加一个新功能：

1. **读取输入** -- 从玩家那里获取猜测值并打印出来
2. **添加循环** -- 让玩家反复猜测并获得反馈
3. **随机数** -- 每局游戏选一个不同的秘密数字
4. **错误处理** -- 优雅地处理无效（非数字）输入

## 添加循环

现在玩家只能猜一次，程序就退出了。真正的游戏需要不断询问，直到玩家猜对为止。我们还需要把字符串输入转换为数字，以便与秘密数字进行比较，并给出"太大"或"太小"的反馈。

<Listing number="5-2" file-name="main.at" caption="循环直到用户猜对">

```auto
fn main() {
    var secret = 42
    print("Guess the number!")
    for true {
        var guess_str = input()
        var guess = parse_int(guess_str)
        print("You guessed: " + guess)
        if guess == secret {
            print("You win!")
            break
        }
        if guess < secret {
            print("Too small!")
        } else {
            print("Too big!")
        }
    }
}
```

### `for true` 无限循环

```auto
for true {
    // ...
}
```

`for true` 创建一个无限循环——它会一直运行，直到循环内部有代码显式停止它。在 Auto 中，`for` 是唯一的循环关键字，它处理所有循环模式。`for true` 的意思是"永远循环"。转译器将其映射为 Rust 的 `loop`、Python 的 `while True`、C 的 `while (1)`，以及 TypeScript 的递归函数。

### `parse_int` 函数

```auto
var guess = parse_int(guess_str)
```

`input()` 返回的是 `str`，但我们需要一个数字才能与秘密数字比较。`parse_int` 将字符串转换为整数。如果字符串是 `"76"`，它返回整数 `76`。编译器推断 `guess` 的类型为 `int`。

注意：在这个版本中，如果玩家输入的不是数字（比如 `"hello"`），`parse_int` 会导致程序崩溃。我们会在最终版本中修复这个问题。

### 比较运算符和 `if`/`else`

```auto

    print("Too small!")
} else {
    print("Too big!")
}
```

我们用 `==` 判断相等，用 `<` 判断小于。`if` 语句在条件为 `true` 时执行其代码块。`else` 块在 `if` 条件为 `false` 时执行。

`break` 关键字立即退出循环。当玩家猜对时，我们打印"You win!"，然后执行 `break`，跳转到 `for` 块之后的代码——由于后面没有代码了，程序结束。

## 错误处理

如果玩家输入的不是数字会怎样？目前 `parse_int("hello")` 会导致程序崩溃。这可不是好的用户体验。我们应该检测到无效输入，打印一条消息，然后让玩家重新输入。

在 Auto 中，`parse_int` 返回一个可选整数：`?int`。可选值要么是一个真正的整数，要么是 `none`（表示"没有值"）。我们可以检查 `none` 来处理错误情况。

<Listing number="5-4" file-name="main.at" caption="带输入验证的完整猜数字游戏">

```auto
fn main() {
    var secret = random_int(1, 100)
    var attempts = 0
    print("Guess the number between 1 and 100!")
    for true {
        print("Please input your guess.")
        var guess_str = input()
        var guess = parse_int(guess_str)
        if guess == none {
            print("Please enter a valid number!")
            continue
        }
        attempts = attempts + 1
        if guess == secret {
            print("You win! You got it in " + attempts + " tries!")
            break
        }
        if guess < secret {
            print("Too small!")
        } else {
            print("Too big!")
        }
    }
}
```

```rust
// Auto-generated by a2r transpiler

#[allow(unused_imports)]
use auto_lang::a2r_std::*;
use rand::Rng;

fn main() {
    let mut secret: i64 = rand::thread_rng().gen_range(1..=100);
    let mut attempts: i64 = 0;
    println!("Guess the number between 1 and 100!");
    loop {
        println!("Please input your guess.");
        let mut guess_str: String = read_line();
        let guess: Option<i64> = guess_str.trim().parse().ok();
        if guess.is_none() {
            println!("Please enter a valid number!");
            continue;
        }
        let mut guess: i64 = guess.unwrap();
        attempts = attempts + 1;
        if guess == secret {
            println!("You win! You got it in {} tries!", attempts);
            break;
        }
        if guess < secret {
            println!("Too small!");
        } else {
            println!("Too big!");
        }
    }
}
```

### 可选类型和 `none`

```auto
var guess = parse_int(guess_str)
if guess == none {
    print("Please enter a valid number!")
    continue
}
```

`parse_int` 现在返回 `?int`（读作"可选 int"）而不是普通的 `int`。可选类型意味着值要么是一个整数，要么是 `none`。当玩家输入 `"hello"` 时，`parse_int` 返回 `none`——没有有效的整数可以提取。当玩家输入 `"42"` 时，它返回 `42`。

我们用 `guess == none` 检测错误情况。如果输入无效，我们打印一条消息，然后用 `continue` 跳过循环体的剩余部分，开始下一次迭代，再次要求玩家输入。

> 在第二阶段（Phase 2）中，你将学习 Auto 的完整错误处理机制——`!T` 类型和
> 解包运算符。现在，用 `== none` 检查是处理可选值的基本方法。

### `continue` 关键字

`continue` 跳过当前循环迭代的剩余部分，回到条件检查。结合 `for true`，它实际上就是"从循环顶部重新开始"。这非常适合跳过无效输入，而不把它算作一次猜测。

### 计数尝试次数

```auto
var attempts = 0
// ...
attempts = attempts + 1
```

我们添加了一个 `attempts` 计数器，从 `0` 开始，每次玩家输入有效猜测时加 `1`。当玩家获胜时，我们在胜利消息中包含尝试次数。注意我们只在 `none` 检查之后才递增 `attempts`，这样无效输入不会被计入。

看看每种目标语言如何处理"无效输入"检查：

- **Rust** 使用 `Option<i64>`，通过 `.ok()`（将 `Result` 转换为 `Option`），
  `.is_none()` 进行检查，`.unwrap()` 提取值。
- **Python** 使用 `try/except ValueError` 块。如果 `int()` 对无效输入失败，
  会抛出 `ValueError`，`except` 分支来处理。
- **C** 使用 `strtoll()`，它会设置 `endptr` 指向第一个非数字字符。如果
  `*endptr != '\0'`，说明输入包含非数字字符。
- **TypeScript** 在 `parseInt()` 之后检查 `isNaN(guess)`。如果输入不是有效数字，
  `parseInt` 返回 `NaN`（Not a Number）。

## 总结

恭喜你——你已经在 Auto 中构建了一个完整的交互式游戏！这个项目综合运用了你在第一阶段学到的所有知识：

- **变量**（`var`）—— `secret`、`attempts`、`guess`、`guess_str` 都使用 `var` 来存储游戏中会变化的值
- **函数**（`fn`）—— 整个游戏都在 `main()` 中，我们还调用了内置函数 `print()`、`input()`、`parse_int()` 和 `random_int()`
- **运算符** —— `==`、`<`、`+` 用于比较和拼接
- **控制流** —— `if`/`else` 用于分支判断，`for true` 用于循环，`break` 退出循环，`continue` 跳过迭代
- **类型** —— `int`、`str`、`?int`（可选整数），编译器从上下文推断类型
- **集合** —— 虽然这个游戏没有直接使用，但你在第四章已经学习了数组、对象、映射和集合
