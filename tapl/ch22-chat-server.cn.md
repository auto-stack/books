# 项目：多人聊天服务器

这是第三阶段的毕业设计，也是本书的最后一章。在过去的二十一章中，你学习了
变量、函数、类型、枚举、模式匹配、错误处理、泛型、闭包、迭代器、Actor、并发、
元编程以及标准库。现在，是时候将所有这些技能整合起来，构建一个真实的项目：
多人聊天服务器。

聊天服务器是一个接受多个用户连接、在他们之间转发消息并管理聊天室的程序。
它是本书每个主要概念交汇处的项目。你需要类型来建模消息，枚举来表示协议，
Actor 来处理并发连接，Map 来跟踪状态，错误处理来应对断线，以及标准库来执行

我们将按以下步骤逐步构建：

1. **消息类型** —— 将协议定义为枚举
2. **服务器 Actor** —— 管理客户端的中央协调器
3. **客户端处理器** —— 每个连接的输入读取逻辑
4. **聊天室** —— 支持加入和离开的多房间频道
5. **主入口** —— 将所有组件组装起来
6. **完整服务器** —— 组装后的完整程序

## 设计概述

在编写任何代码之前，让我们先思考架构。聊天服务器有三个主要组件：

1. **服务器 Actor** —— 一个维护全局状态的单一 Actor：哪些用户已连接、
   哪些房间存在、谁在哪个房间里。所有消息都通过这个 Actor 流转。

2. **客户端处理器** —— 每个已连接用户一个 Actor。每个处理器从其用户读取
   输入，解析命令，并将消息转发给服务器 Actor。

3. **消息协议** —— 一个枚举，定义系统处理的每种消息类型：连接、断开、
   聊天、私信、加入房间、离开房间、房间消息和系统通知。

这个设计遵循了[第 15 章][ch15]的 Actor 模型。服务器是唯一的协调点。客户端
处理器是独立的 Actor，通过消息与服务器通信。没有共享内存，没有锁，没有
数据竞争 —— 只有 Actor 之间发送消息。

让我们分析一下设计决策：

1. **`Connect(String)`** —— 携带用户名。当用户连接时，服务器注册他们。

2. **`Disconnect(String)`** —— 携带用户名。服务器将他们从所有房间移除并
   通知其他人。

3. **`Chat(String, String)`** —— 携带发送者名称和消息文本。这是广播给所有
   用户的公开消息。

4. **`PrivateMsg(String, String, String)`** —— 携带发送者、接收者和文本。只有
   接收者能看到此消息。

5. **`System(String)`** —— 来自服务器本身的通知："Alice 加入"、"Bob 离开"等。

为什么用枚举？因为编译器强制我们处理每个变体。如果稍后添加新的消息类型
（比如 `Kick(String, String)`），编译器会在每个未处理它的 `is` 表达式处
发出警告。这就是我们在[第 7 章][ch07]学到的穷尽模式匹配。

在 C 中，这映射为带有手动 `switch` 语句的标记联合。在 Python 中，映射为
带有 `isinstance` 检查的数据类联合。在 TypeScript 中，映射为对 `kind` 字段
进行 `switch` 的判别联合。在所有情况下，模式都是相同的：一个类型，多个变体，
穷尽处理。

// Listing 22-02: Server actor that manages connected clients

enum Message {
    Connect(String),
    Disconnect(String),
    Chat(String, String),
    PrivateMsg(String, String, String),
    System(String)
}

fn server() {
    var clients: Map<String, int> = {}
    for true {
        on msg Message {
            is msg {
                Connect(name) => {
                    clients[name] = 1
                    println("[SYS] " + name + " joined. Online: " + clients.len())
                },
                Disconnect(name) => {
                    clients.remove(name)
                    println("[SYS] " + name + " left. Online: " + clients.len())
                },
                Chat(from, text) => {
                    println(from + ": " + text)
                },
                PrivateMsg(from, to, text) => {
                    println("[PM] " + from + " -> " + to + ": " + text)
                },
                System(text) => {
                    println("[SYS] " + text)
                }
            }
        }
    }
}

fn main() {
    spawn server
    send server Message.Connect("Alice".to_string())
    send server Message.Connect("Bob".to_string())
    send server Message.Chat("Alice".to_string(), "Hello Bob!".to_string())
    send server Message.PrivateMsg("Bob".to_string(), "Alice".to_string(), "Hi Alice!".to_string())
    send server Message.Disconnect("Bob".to_string())
}

typedef enum MessageKind {
    MsgKind_Connect,
    MsgKind_Disconnect,
    MsgKind_Chat,
    MsgKind_PrivateMsg,
    MsgKind_System,
} MessageKind;

typedef struct Message {
    MessageKind kind;
    const char* field1;
    const char* field2;
    const char* field3;
} Message;

typedef struct {
    char names[MAX_CLIENTS][64];
    int count;
} ClientList;

ClientList clients = { .count = 0 };

void server_handle(Message* msg) {
    switch (msg->kind) {
        case MsgKind_Connect: {
            strncpy(clients.names[clients.count], msg->field1, 63);
            clients.names[clients.count][63] = '\0';
            clients.count++;
            printf("[SYS] %s joined. Online: %d\n", msg->field1, clients.count);
            break;
        }
        case MsgKind_Disconnect: {
            int found = -1;
            for (int i = 0; i < clients.count; i++) {
                if (strcmp(clients.names[i], msg->field1) == 0) {
                    found = i;
                    break;
                }
            }
            if (found >= 0) {
                for (int i = found; i < clients.count - 1; i++) {
                    strncpy(clients.names[i], clients.names[i + 1], 63);
                }
                clients.count--;
            }
            printf("[SYS] %s left. Online: %d\n", msg->field1, clients.count);
            break;
        }
        case MsgKind_Chat:
            printf("%s: %s\n", msg->field1, msg->field2);
            break;
        case MsgKind_PrivateMsg:
            printf("[PM] %s -> %s: %s\n", msg->field1, msg->field2, msg->field3);
            break;
        case MsgKind_System:
            printf("[SYS] %s\n", msg->field1);
            break;
    }
}

服务器 Actor 维护一个 `clients` 映射来跟踪在线用户。`is` 表达式中的每个
变体处理不同的消息类型：

- **`Connect`** 将用户添加到 `clients` 映射中并宣布其到来。
- **`Disconnect`** 将其移除并宣布其离开。
- **`Chat`** 打印消息（在真实服务器中，它会广播给所有客户端）。
- **`PrivateMsg`** 打印私信标识。
- **`System`** 打印服务器通知。

关键洞察是，所有状态变更都发生在单一 Actor 内部。`clients` 映射是
`server` 函数私有的 —— 没有其他 Actor 可以访问它。这消除了数据竞争，
不需要任何锁或互斥量。Actor 模型保证一次只处理一条消息。

在 Rust 中，服务器在带 `mpsc::Receiver` 的生成线程中运行。在 Python 中，
在带 `queue.Queue` 的守护线程中运行。在 C 中，我们用消息数组模拟消息循环。
在 TypeScript 中，同步处理数组中的消息。Auto 版本最简洁：`spawn server`，
`send server Message.Connect(...)`。两行代码。

    Connect(String),
    Disconnect(String),
    Chat(String, String),
    PrivateMsg(String, String, String),
    System(String),
    JoinRoom(String, String),
    LeaveRoom(String, String),

    var clients: Map<String, int> = {}
    var rooms: Map<String, List<String>> = {
        "general": [],
        "random": []
    }
    for true {
        on msg Message {
            is msg {
                Connect(name) => {
                    clients[name] = 1
                    rooms["general"].push(name.to_string())
                    println("[SYS] " + name + " joined. Online: " + clients.len())
                },
                Disconnect(name) => {

                        room.value.remove(name)
                    }
                    println("[SYS] " + name + " left. Online: " + clients.len())

                        if !rooms[room_name].contains(name) {
                            rooms[room_name].push(name.to_string())
                            println("[SYS] " + name + " joined #" + room_name)
                        }
                    } else {
                        println("[SYS] Room #" + room_name + " does not exist")
                    }
                },
                LeaveRoom(name, room_name) => {
                    rooms[room_name].remove(name)
                    println("[SYS] " + name + " left #" + room_name)
                },
                RoomMsg(room_name, from, text) => {
                    var members = rooms[room_name]
                    println("[#" + room_name + "] " + from + ": " + text)
                    println("  (delivered to " + members.len() + " members)")
                },
                Chat(from, text) => {
                    println(from + ": " + text)
                },
                PrivateMsg(from, to, text) => {
                    println("[PM] " + from + " -> " + to + ": " + text)
                },
                System(text) => {
                    println("[SYS] " + text)
                }
            }
        }
    }
}

    | { kind: "System"; field1: string }
    | { kind: "JoinRoom"; field1: string; field2: string }
    | { kind: "LeaveRoom"; field1: string; field2: string }
    | { kind: "RoomMsg"; field1: string; field2: string; field3: string };

function server(messages: Message[]): void {
    const clients = new Map<string, number>();
    const rooms = new Map<string, string[]>();
    rooms.set("general", []);
    rooms.set("random", []);

    for (const msg of messages) {
        switch (msg.kind) {
            case "Connect": {
                clients.set(msg.field1, 1);
                rooms.get("general")!.push(msg.field1);
                console.log(`[SYS] ${msg.field1} joined. Online: ${clients.size}`);
                break;
            }
            case "Disconnect": {
                clients.delete(msg.field1);
                for (const [, members] of rooms) {
                    const idx = members.indexOf(msg.field1);
                    if (idx >= 0) members.splice(idx, 1);
                }
                console.log(`[SYS] ${msg.field1} left. Online: ${clients.size}`);
                break;
            }
            case "JoinRoom": {
                const members = rooms.get(msg.field2);
                if (members) {
                    if (!members.includes(msg.field1)) {
                        members.push(msg.field1);
                        console.log(`[SYS] ${msg.field1} joined #${msg.field2}`);
                    }
                } else {
                    console.log(`[SYS] Room #${msg.field2} does not exist`);
                }
                break;
            }
            case "LeaveRoom": {
                const members = rooms.get(msg.field2);
                if (members) {
                    const idx = members.indexOf(msg.field1);
                    if (idx >= 0) members.splice(idx, 1);
                }
                console.log(`[SYS] ${msg.field1} left #${msg.field2}`);
                break;
            }
            case "RoomMsg": {
                const members = rooms.get(msg.field1) || [];
                console.log(`[#${msg.field1}] ${msg.field2}: ${msg.field3}`);
                console.log(`  (delivered to ${members.length} members)`);
                break;
            }
            case "Chat":
                console.log(`${msg.field1}: ${msg.field2}`);
                break;
            case "PrivateMsg":
                console.log(`[PM] ${msg.field1} -> ${msg.field2}: ${msg.field3}`);
                break;
            case "System":
                console.log(`[SYS] ${msg.field1}`);
                break;
        }
    }
}

function main(): void {
    const messages: Message[] = [
        { kind: "Connect", field1: "Alice" },
        { kind: "Connect", field1: "Bob" },
        { kind: "JoinRoom", field1: "Alice", field2: "random" },
        { kind: "JoinRoom", field1: "Bob", field2: "random" },
        { kind: "RoomMsg", field1: "random", field2: "Alice", field3: "Anyone here?" },
        { kind: "RoomMsg", field1: "random", field2: "Bob", field3: "Yes!" },
        { kind: "LeaveRoom", field1: "Alice", field2: "random" },
    ];
    server(messages);
}

## 进阶方向

这个聊天服务器是一个坚实的基础。以下是一些扩展思路：

1. **身份验证** —— 添加 `Login(String, String)` 消息，在允许连接前检查凭据。
   使用 `std::crypto` 存储密码哈希。

2. **持久化** —— 使用 `std::fs` 的 `write_file` 将聊天记录保存到文件。服务器

3. **WebSocket 支持** —— 使用 `std::net` 接受 WebSocket 连接而非原始 TCP，

4. **管理员命令** —— 添加 `/kick`、`/ban` 和 `/create-room` 命令，仅限
   管理员使用。

5. **消息历史** —— 为每个房间保存最近 N 条消息，新用户加入时发送给他们。

6. **输入指示器** —— 添加 `Typing(String, String)` 消息，向房间其他成员显示
   "Alice 正在输入..."。

7. **速率限制** —— 限制用户每秒可以发送的消息数量以防止垃圾信息。

8. **加密** —— 使用 `std::crypto` 对私信进行端到端加密，使服务器也无法读取。

嵌入式系统、数据管道还是操作系统，这里学到的模式都将为你提供帮助。

感谢你的阅读。现在，去构建一些了不起的东西吧。

## 总结

这个毕业设计将第三阶段和整本书的每个概念整合在一起：

| 概念 | 使用位置 |
|------|----------|
| 枚举 | `Message` 枚举定义协议 |
| 模式匹配 | `is msg { Connect(name) => ... }` |
| Map | `clients: Map<String, int>`、`rooms: Map<String, List<String>>` |
| List | 房间成员列表、消息分片 |
| 字符串方法 | `split`、`join`、`starts_with`、`contains`、`to_string` |
| Actor | `spawn server`、`spawn client_handler` |
| 消息传递 | `send server Message.Connect(...)` |
| `on` 接收 | `on msg Message { ... }`、`on input String { ... }` |
| 错误处理 | 用户名冲突检查、缺失用户检查 |
| 标准库 | `std::net`、`format!`、`println` |
| 函数 | `server`、`client_handler`、`broadcast`、`broadcast_room` |
| 类型 | `Map<String, List<String>>`、`Message` 变体 |

核心要点：

1. **先设计协议** —— `Message` 枚举是所有组件之间的契约。把它设计好，
   其他一切都会顺畅。

2. **一个 Actor 拥有状态** —— 服务器 Actor 是客户端和房间的唯一真相来源。
   这消除了整类并发 bug。

3. **逐步构建** —— 我们从消息类型开始，然后是服务器，然后是客户端处理器，
   接着是房间，最后是完整程序。每一步都可以独立测试。

4. **Actor 模型天然可扩展** —— 添加更多功能（认证、持久化、管理命令）意味着
   添加更多消息类型和更多处理程序。架构保持不变。

5. **错误处理不是事后补丁** —— 我们检查了重复的用户名、缺失的用户和不存在
   的房间。这些检查是正常控制流的一部分，而非事后添加。
