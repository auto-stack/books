# 集合与节点

到目前为止，我们创建的每个变量都只保存一个值——一个数字、一个字符串、一个布尔值。但真正的程序处理的是一组组事物：一组名字、一张成绩表、一组唯一 ID。Auto 提供了多种集合类型，让你可以存储、查询和转换多个值。在本章中，我们还将认识 `node`——Auto 中用于构建链式结构（如链表和树）的关键字。

在本章中，我们将讨论：

- **数组**——使用 `[1, 2, 3]` 字面量语法创建的有序序列
- **对象**——使用 `{ name: "Alice", age: 30 }` 创建的命名字段记录
- **HashMap**——从字符串键字面量推断出的键值映射
- **Set**——使用 `{1, 2, 3}` 创建的去重集合
- **列表操作**——`push`、`pop`、`contains`、`sort` 等
- **字符串作为集合**——`len`、索引、`split`、`to_upper`
- **节点**——使用 `node`、`?Item` 和 `none` 构建链式结构

## 数组

_数组_ 是一个有序的、可索引的值集合。你用方括号和逗号分隔的元素列表来创建它：

```auto
var numbers = [1, 2, 3, 4, 5]
```

类型从元素中推断——因为这里的每个值都是 `int`，所以 `numbers` 的类型是 `[int]`。你可以用 `numbers[0]` 通过从零开始的索引访问元素，用 `numbers.len()` 获取长度。你可以用 `for item in array` 遍历数组。

<Listing number="4-1" file-name="main.at" caption="创建数组、索引访问和遍历">

```auto
fn main() {
    var numbers = [1, 2, 3, 4, 5]
    print(numbers[0])
    print(numbers.len())
    var names = ["Alice", "Bob", "Charlie"]
    for name in names {
        print(name)
    }
}
```

## 对象

_对象_ 是一组命名字段的集合。你用花括号和 `key: value` 对来创建它：

```auto
var person = { name: "Alice", age: 30 }
```

类型从字段中推断——`person` 有 `name`（类型 `str`）和 `age`（类型 `int`）两个字段。你用点号访问字段（`person.name`），用点号赋新值（`person.age = 31`）。因为我们用 `var` 声明了 `person`，所以对象是可变的。

fn main() {

    person.age = 31
    print(person.age)
}

每种语言处理类对象数据的方式不同：

- **Rust** 使用 `AutoObject`，一个动态键值存储。字段通过 `.set()` 和 `.get()` 设置和获取，值包装在 `AutoValue` 中。

## HashMap

当对象字面量使用字符串键时，Auto 会推断为 _HashMap_ 而不是普通对象：

```auto
var scores = { "Alice": 90, "Bob": 85, "Charlie": 92 }
```

与普通对象的区别在于，你用括号记法访问值（`scores["Alice"]`），而且映射可以在运行时增长——你可以插入新的键。类型被推断为 `Map<str, int>`。

<Listing number="4-3" file-name="main.at" caption="从字符串键字面量推断 HashMap">

```auto
fn main() {
    var scores = { "Alice": 90, "Bob": 85, "Charlie": 92 }
    print(scores["Alice"])
    scores["Dave"] = 88
    print(scores.len())
}
```

#include <stdio.h>
#include <string.h>
#include <stdlib.h>

typedef struct MapEntry {
    char key[256];
    long long value;
    struct MapEntry* next;
} MapEntry;

typedef struct {
    MapEntry* head;
    int size;
} HashMap;

void map_insert(HashMap* map, const char* key, long long value) {
    MapEntry* entry = map->head;
    while (entry != NULL) {
        if (strcmp(entry->key, key) == 0) {
            entry->value = value;
            return;
        }
        entry = entry->next;
    }
    MapEntry* new_entry = (MapEntry*)malloc(sizeof(MapEntry));
    strcpy(new_entry->key, key);
    new_entry->value = value;
    new_entry->next = map->head;
    map->head = new_entry;
    map->size++;
}

long long map_get(HashMap* map, const char* key) {
    MapEntry* entry = map->head;
    while (entry != NULL) {
        if (strcmp(entry->key, key) == 0) {
            return entry->value;
        }
        entry = entry->next;
    }
    return 0;
}

int main(void) {
    HashMap scores = {NULL, 0};
    map_insert(&scores, "Alice", 90);
    map_insert(&scores, "Bob", 85);
    map_insert(&scores, "Charlie", 92);
    printf("%lld\n", map_get(&scores, "Alice"));
    map_insert(&scores, "Dave", 88);
    printf("%d\n", scores.size);
    return 0;
}

Auto 中对象和 HashMap 的关键区别在于 _键类型_。对象使用标识符键（`{ name: "Alice" }`），通过点号访问。HashMap 使用字符串键（`{ "Alice": 90 }`），通过括号访问。编译器根据语法来判断你要的是哪一种。

看看每种语言如何处理这种情况：

- **Rust** 使用 `std::collections::HashMap<String, i64>`，配合 `.insert()` 和 `.get()`。`.get()` 上的 `.unwrap()` 在这里是安全的，因为我们知道键存在。
- **Python** 使用普通的 `dict`——与 Auto 语法相同。括号访问和 `len()` 自然可用。
- **C** 没有内置的哈希映射，所以转译器生成了一个基于链表的映射，包含 `map_insert` 和 `map_get` 辅助函数。
- **TypeScript** 使用索引签名 `{ [key: string]: number }` 和 `Object.keys()` 获取长度。

## Set

_Set_ 是一个唯一值集合。你用花括号和逗号分隔的值来创建它——语法与对象相同，但元素不是字符串：

```auto
var unique = {1, 2, 3, 2, 1}
```

重复值会自动被丢弃。编译器推断为 `Set<int>`，因为元素是整数。你可以用 `.add()` 添加元素，用 `.contains()` 检查成员。

<Listing number="4-4" file-name="main.at" caption="Set 字面量的去重和成员检查">

```auto
fn main() {
    var unique = {1, 2, 3, 2, 1}
    print(unique.len())
    unique.add(4)
    print(unique.contains(3))
}
```

typedef struct {
    SetEntry* head;

} HashSet;

void set_insert(HashSet* set, long long value) {

    while (entry != NULL) {
        if (entry->value == value) {
            return;

        entry = entry->next;

注意语言差异：

- **Rust** 使用 `std::collections` 中的 `HashSet<i64>`。`.insert()` 方法会静默忽略重复项，`.contains()` 接受一个引用。
- **Python** 使用原生 `set` 字面量。`in` 运算符替代了 Auto 的 `.contains()` 方法。
- **C** 生成了一个基于链表的集合，配有辅助函数。插入前会检查重复。

Auto 如何区分 Set `{1, 2, 3}`、对象 `{ name: "Alice" }` 和 HashMap `{ "Alice": 90 }`？通过 _键_ 的形式：

- **标识符键**（无引号）：`{ name: "Alice" }` 是**对象**。
- **字符串键**（有引号）：`{ "Alice": 90 }` 是 **HashMap**。
- **无键**（只有值）：`{1, 2, 3}` 是 **Set**。

编译器使用这些语法线索来推断正确的类型。你永远不需要显式写 `HashMap<String, i64>` 或 `HashSet<int>`——Auto 会自己搞定。

#include <stdio.h>
#include <stdlib.h>
#include <stdbool.h>

typedef struct {
    long long* data;
    int size;
    int capacity;
} Vec;

void vec_push(Vec* v, long long value) {
    if (v->size >= v->capacity) {
        v->capacity = v->capacity == 0 ? 4 : v->capacity * 2;
        v->data = (long long*)realloc(v->data, v->capacity * sizeof(long long));
    }
    v->data[v->size++] = value;
}

long long vec_pop(Vec* v) {
    return v->data[--v->size];
}

bool vec_contains(Vec* v, long long value) {
    for (int i = 0; i < v->size; i++) {
        if (v->data[i] == value) return true;
    }
    return false;
}

int compare(const void* a, const void* b) {
    long long diff = *(long long*)a - *(long long*)b;
    return (diff > 0) - (diff < 0);
}

void vec_print(Vec* v) {
    printf("[");
    for (int i = 0; i < v->size; i++) {
        if (i > 0) printf(", ");
        printf("%lld", v->data[i]);
    }
    printf("]\n");
}

int main(void) {
    Vec list = {NULL, 0, 0};
    vec_push(&list, 10);
    vec_push(&list, 20);
    vec_push(&list, 30);
    vec_push(&list, 40);
    vec_print(&list);
    long long last = vec_pop(&list);
    printf("%lld\n", last);
    printf("%s\n", vec_contains(&list, 20) ? "true" : "false");
    qsort(list.data, list.size, sizeof(long long), compare);
    vec_print(&list);
    free(list.data);
    return 0;
}

字符串方法在所有四种目标语言中都是一致的，但细节有所不同：

- **Rust** 使用 `.len()` 获取字节长度，`.chars().nth(0)` 访问字符（因为 Rust
  字符串是 UTF-8 编码的），`.split(", ").collect()` 拆分，`.to_uppercase()`
  转换大写。
- **Python** 使用内置函数 `len(s)`，`s[0]` 索引访问，`s.split(", ")` 拆分，
  `s.upper()` 转大写。
- **C** 使用 `strlen()` 获取长度，直接数组索引 `s[0]`，`strtok()` 拆分，手动
  循环配合 `toupper()` 转换大写。
- **TypeScript** 使用 `.length`（属性，不是方法），`s[0]` 索引访问，
  `.split(", ")` 拆分，`.toUpperCase()` 转大写。

## 节点

数组、对象和映射都是 _连续_ 数据结构——它们的元素在内存中相邻存储。有时候你需要一种 _链式_ 结构，其中每个元素指向下一个元素。想象一条链条：每个环节连接到下一个环节，最后一个环节什么都不连接。

Auto 提供了 `node` 关键字来实现这种模式。节点类似于轻量级的结构体，其中一个或多个字段可以是 _可选的_——它们可能指向另一个节点，也可能是 `none`。

### 定义节点

你用 `node` 关键字、字段名和类型来定义一个节点。`?` 前缀表示"可选"——值可以是一个实际实例，也可以是 `none`：

```auto
node Item {
    value int
    next ?Item
}
```

这定义了一个名为 `Item` 的节点，包含两个字段：`value`（整数）和 `next`（另一个 `Item` 或 `none`）。`?Item` 类型是 Auto 表达"可空 Item"的方式——类似于 Rust 中的 `Option<Item>` 或 TypeScript 中的 `Item | null`。

<Listing number="4-7" file-name="main.at" caption="定义节点类型并构建链式结构">

```auto
node Item {
    value int
    next ?Item
}

fn main() {
    var third = Item { value: 30, next: none }
    var second = Item { value: 20, next: third }
    var first = Item { value: 10, next: second }
    print(first.value)
    print(first.next.value)
}
```

  `first.next!.value` 告诉 TypeScript 该处的值不为 null。

### 遍历节点链

访问 `first.next.value` 对于两层深度可以工作，但真正的链式结构可以任意长。标准模式是从头部开始，逐个走过每个节点，直到到达 `none`：

<Listing number="4-8" file-name="main.at" caption="用条件循环遍历链式结构">

```auto
node Item {
    value int
    next ?Item
}

fn main() {
    var third = Item { value: 30, next: none }
    var second = Item { value: 20, next: third }
    var first = Item { value: 10, next: second }
    var current = first
    for current != none {
        print(current.value)
        current = current.next
    }
}
```

遍历模式在每种语言中都是相同的：

1. 从 `current = first`（链的头部）开始。
2. 当 `current` 不为 `none` / `NULL` / `null` 时循环。
3. 打印 `current.value`。
4. 移动到下一个节点：`current = current.next`。

在 Auto 中，`for current != none { ... }` 是一个条件循环——只要条件为真就持续运行。转译器将其转换为每种目标语言中的 `while`，因为 `for cond` 是 Auto 书写其他语言中 `while` 的方式。

注意遍历代码在所有五种语言中几乎完全相同。链表遍历是那种 _逻辑_ 通用但只有"是否为空"的语法不同的模式之一：

| 语言 | 空值检查 | 空值 |
|------|---------|------|
| Auto | `current != none` | `none` |
| Rust | `current.is_some()` | `None` |
| Python | `current is not None` | `None` |
| C | `current != NULL` | `NULL` |
| TypeScript | `current !== null` | `null` |

## 总结

在本章中，你学习了如何在 Auto 中使用集合和链式结构：

- **数组** `[1, 2, 3]` 存储有序序列。通过 `arr[0]` 索引访问，通过
  `arr.len()` 获取长度，通过 `for item in arr` 遍历。
- **对象** `{ name: "Alice" }` 存储命名字段。通过点号访问（`obj.name`），通过
  赋值修改（`obj.age = 31`）。
- **HashMap** `{ "Alice": 90 }` 从字符串键对象字面量推断。通过括号访问
  （`map["Alice"]`），通过括号赋值插入。
- **Set** `{1, 2, 3}` 存储唯一值。通过 `.add()` 添加，通过 `.contains()`
  检查成员。
- **列表操作**——`.push()`、`.pop()`、`.contains()`、`.sort()`——让你原地
  操作数组。
- **字符串作为集合**——`.len()`、`s[0]`、`.split()`、`.to_upper()`——将
  字符串视为字符序列。
- **节点** 使用 `node` 和 `?T` 构建链式结构。`none` 标记链的末端。遍历使用
  `for current != none { ... }`。

编译器从语法中推断集合类型——你永远不需要写 `Vec<i64>`、`HashMap<String,
i64>` 或 `HashSet<i64>`。Auto 从字面量中读取你的意图，选择正确的类型。在
下一章中，我们将了解如何用 `type` 定义自己的自定义类型，并用 `ext` 构建方法。
