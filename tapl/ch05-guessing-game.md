# Project: Guessing Game

This is the capstone project for Phase 1. Over the first four chapters, you've
learned about variables, operators, functions, collections, and control flow.
Now it's time to put all of those skills together by building a complete
interactive program: a number guessing game.

Here's how the game works: the program picks a secret number between 1 and 100.
The player types in a guess, and the program says "Too small!" or "Too big!"
until the player gets it right. We'll build this in four stages, adding one
feature at a time:

1. **Reading input** -- get a guess from the player and print it
2. **Adding a loop** -- let the player guess repeatedly with feedback
3. **Random numbers** -- pick a different secret number each game
4. **Error handling** -- deal with invalid (non-number) input gracefully

Let's walk through this line by line.

### The `print` Function

```auto
print("Guess the number!")
```

`print` is a built-in function that writes text to the terminal. It takes a
single `str` argument and adds a newline at the end. We use it twice: once to
announce the game, and once to prompt the player.

### The `input` Function

```auto
var guess = input()
```

`input()` is a built-in function that reads one line of text from the player
and returns it as a `str`. The program pauses here and waits for the user to
type something and press Enter. We store the result in `guess`, declared with
`var` because we'll be reassigning it in later versions.

### String Concatenation with `+`

```auto
print("You guessed: " + guess)
```

The `+` operator concatenates two strings. Auto infers that the result is also a
`str`. If you wrote `"a" + 42`, the compiler would report a type error -- you
need to convert the number to a string first. We'll see how to do that with
`parse_int` in the next section.

### Running the Program

```console
$ automan run
Guess the number!
Please input your guess.
76
You guessed: 76
```

The program prints the prompts, waits for your input, and echoes it back. Not
much of a game yet, but the foundation is in place.

Notice how each target language handles user input differently:

- **Rust** uses a custom `read_line()` function. The result is a `String` that
  the transpiler manages.
- **Python** maps directly to the built-in `input()` function. This is the
  closest match -- Python's `input()` does exactly what Auto's `input()` does.
- **C** uses `fgets()` to read into a fixed-size buffer, then strips the
  trailing newline with `strcspn`. Manual memory management is required.
- **TypeScript** uses the `readline` module with a callback-based API. The
  callback receives the user's input as a `string`.

## Adding a Loop

Right now the player gets one guess and the program exits. A real game needs to
keep asking until the player wins. We also need to convert the string input to a
number so we can compare it with the secret number, and give feedback about
whether the guess is too high or too low.

<Listing number="5-2" file-name="main.at" caption="Looping until the user guesses correctly">

```auto
fn main() {
    var secret = 42
    print("Guess the number!")
    for true {
        var guess_str = input()
        var guess = parse_int(guess_str)
        print("You guessed: " + guess)
        if guess == secret {
            print("You win!")
            break
        }
        if guess < secret {
            print("Too small!")
        } else {
            print("Too big!")
        }
    }
}
```

There's a lot of new material here. Let's break it down.

### The `for true` Infinite Loop

```auto
for true {
    // ...
}
```

`for true` creates an infinite loop -- it runs until something inside it
explicitly stops it. In Auto, `for` is the only loop keyword, and it handles
all looping patterns. `for true` means "loop forever." The transpiler maps this
to `loop` in Rust, `while True` in Python, `while (1)` in C, and a recursive
function in TypeScript.

### The `parse_int` Function

```auto
var guess = parse_int(guess_str)
```

`input()` returns a `str`, but we need a number to compare with the secret.
`parse_int` converts a string to an integer. If the string is `"76"`, it returns
the integer `76`. Auto infers that `guess` has type `int`.

Note: in this version, if the player types something that isn't a number (like
`"hello"`), `parse_int` will crash the program. We'll fix that in the final
version.

### Comparison Operators and `if`/`else`

```auto
if guess == secret {
    print("You win!")
    break
}
if guess < secret {
    print("Too small!")
} else {
    print("Too big!")
}
```

We use `==` for equality, `<` for less-than. The `if` statement runs its block
when the condition is `true`. The `else` block runs when the `if` condition is
`false`.

The `break` keyword exits the loop immediately. When the player guesses
correctly, we print "You win!" and then `break`, which jumps to the code after
the `for` block -- and since there's nothing after it, the program ends.

## Error Handling

What happens if the player types something that isn't a number? Right now,
`parse_int("hello")` crashes the program. That's not a great user experience.
Instead, we should detect the bad input, print a message, and let the player try
again.

In Auto, `parse_int` returns an optional integer: `?int`. An optional value is
either a real integer or `none` (meaning "no value"). We can check for `none`
to handle the error case.

<Listing number="5-4" file-name="main.at" caption="The complete guessing game with input validation">

```auto
fn main() {
    var secret = random_int(1, 100)
    var attempts = 0
    print("Guess the number between 1 and 100!")
    for true {
        print("Please input your guess.")
        var guess_str = input()
        var guess = parse_int(guess_str)
        if guess == none {
            print("Please enter a valid number!")
            continue
        }
        attempts = attempts + 1
        if guess == secret {
            print("You win! You got it in " + attempts + " tries!")
            break
        }
        if guess < secret {
            print("Too small!")
        } else {
            print("Too big!")
        }
    }
}
```

```rust
// Auto-generated by a2r transpiler

#[allow(unused_imports)]
use auto_lang::a2r_std::*;
use rand::Rng;

fn main() {
    let mut secret: i64 = rand::thread_rng().gen_range(1..=100);
    let mut attempts: i64 = 0;
    println!("Guess the number between 1 and 100!");
    loop {
        println!("Please input your guess.");
        let mut guess_str: String = read_line();
        let guess: Option<i64> = guess_str.trim().parse().ok();
        if guess.is_none() {
            println!("Please enter a valid number!");
            continue;
        }
        let mut guess: i64 = guess.unwrap();
        attempts = attempts + 1;
        if guess == secret {
            println!("You win! You got it in {} tries!", attempts);
            break;
        }
        if guess < secret {
            println!("Too small!");
        } else {
            println!("Too big!");
        }
    }
}
```

This is the final version of the game. Let's look at what's new.

### Optional Types and `none`

```auto
var guess = parse_int(guess_str)
if guess == none {
    print("Please enter a valid number!")
    continue
}
```

`parse_int` now returns `?int` (read "optional int") instead of plain `int`. An
optional type means the value can either be an integer or `none`. When the
player types `"hello"`, `parse_int` returns `none` -- there's no valid integer
to extract. When the player types `"42"`, it returns `42`.

We check `guess == none` to detect the error case. If the input was invalid, we
print a message and use `continue` to skip the rest of the loop body and start
the next iteration, asking the player again.

> In Phase 2, you'll learn about Auto's full error handling with `!T` and the
> unwrap operator. For now, checking `== none` is the basic way to handle
> optional values.

### The `continue` Keyword

`continue` skips the rest of the current loop iteration and goes back to the
condition check. Combined with `for true`, it effectively means "start the loop
over from the top." This is perfect for skipping bad input without counting it
as a guess.

### Counting Attempts

```auto
var attempts = 0
// ...
attempts = attempts + 1
```

We added an `attempts` counter that starts at `0` and increments by `1` each
time the player enters a valid guess. When the player wins, we include the
count in the victory message. Note that we only increment `attempts` _after_
the `none` check, so invalid inputs don't count.

The game handles all the edge cases: non-number input, too-high guesses,
too-low guesses, and correct guesses. It counts attempts, gives feedback, and
exits cleanly when the player wins.

Look at how each target language handles the "invalid input" check:

- **Rust** uses `Option<i64>` with `.ok()` (converts `Result` to `Option`),
  `.is_none()` for the check, and `.unwrap()` to extract the value after the
  check.
- **Python** uses a `try/except ValueError` block. If `int()` fails on bad
  input, it raises `ValueError`, and the `except` branch handles it.
- **C** uses `strtoll()` which sets `endptr` to the first non-numeric
  character. If `*endptr != '\0'`, the input contained non-digit characters.
- **TypeScript** checks `isNaN(guess)` after `parseInt()`. If the input wasn't
  a valid number, `parseInt` returns `NaN` (Not a Number).
